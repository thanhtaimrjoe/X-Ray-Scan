# Release Checklist

## Development

- [ ] Flutter installed and available in PATH.
- [ ] Flutter project created under `app/`.
- [ ] Flame integrated.
- [ ] Core gameplay implemented.
- [ ] Local high score implemented.
- [ ] Sound toggle implemented.
- [ ] AdMob test ads integrated.
- [ ] Unit tests added for game rules.
- [ ] Manual playtest completed on Android device or emulator.

## Google Play Console

- [ ] App created in Play Console.
- [ ] Package name confirmed.
- [ ] App category selected.
- [ ] Contains ads set to Yes.
- [ ] Data Safety completed.
- [ ] Content rating questionnaire completed.
- [ ] Target audience selected carefully.
- [ ] Privacy policy URL added.
- [ ] Store listing written.
- [ ] Screenshots prepared.
- [ ] App icon prepared.
- [ ] Feature graphic prepared.

## AdMob

- [ ] AdMob account ready.
- [ ] Android app created in AdMob.
- [ ] Banner ad unit created.
- [ ] Interstitial ad unit created.
- [ ] Rewarded ad unit created.
- [ ] App linked to Google Play after Play listing exists.
- [ ] Production IDs configured outside Git.

## Android Release

- [ ] Release keystore created and backed up securely.
- [ ] `key.properties` excluded from Git.
- [ ] Release build succeeds.
- [ ] `.aab` generated.
- [ ] Release build tested.
- [ ] Version code and version name set.

## Pre-Submission Policy Check

- [ ] No child-directed marketing.
- [ ] No misleading screenshots.
- [ ] No accidental-click ad placement.
- [ ] No live ad self-clicking.
- [ ] No excessive interstitial frequency.
- [ ] Privacy policy matches actual SDK behavior.

## Launch

- [ ] Internal testing release uploaded.
- [ ] Closed testing completed if required by account type.
- [ ] Production release submitted.
- [ ] Review feedback tracked.
